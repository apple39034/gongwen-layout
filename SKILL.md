# 公文具体排版格式

## 正文要素与字号/字体

- **文件标题**（方正小标宋简体，二号居中）
- **正文**（仿宋三号，行距：固定值 28.95,段落首行缩进2个字符）

## 标题层级示例

- 一、一级标题（黑体三号）
  - （一）二级标题：（楷体三号）
    - 1. 三级标题：（仿宋三号）
      - （1）四级标题：（仿宋三号）
- (空一行)
- 附件：（仿宋三号）
- <空三行>

## 落款与附件

- **落款**：（仿宋三号，靠右，向左四个字符）
- （联系人：）（空两个字）
- 落款包含：发文单位名称 + 成文日期，分两行，均靠右缩进四个字符

- **附件1**（三号黑体）
  - 标题第一行黑体，其他仿宋

## 页面与装订设置

- **页边距**：上下 36mm，左右 27mm
- **版式**：页眉 15mm，页脚 27mm
- **页码**：底端外侧，宋体，四号
- **装订**：平钉，上下 7cm

## 排版注意事项（编程实现时必须遵守）

### 1. 标题自动编号问题
- Word 的 Heading 样式可能携带 `numPr`（自动编号），导致标题前出现多余数字（如 1.2.2）
- 必须从样式定义层面移除 `numPr`，而不仅仅在段落层面处理
- python-docx 实现：遍历 `doc.styles['Heading 1/2/3']._element`，找到 `w:pPr` 下的 `w:numPr` 并删除

### 2. 各级标题缩进
- 一级标题（黑体三号）和二级标题（楷体三号）均须缩进两个中文字
- 实现方式：`pf.left_indent = Pt(32)`（2 × 三号16pt）；`pf.first_line_indent = Pt(0)`
- 注意：使用 `left_indent`（整段左缩进），不是 `first_line_indent`（仅首行缩进）

### 3. 图片段落行距
- 正文固定行距（28.95pt EXACTLY）会裁剪高于该值的图片，导致图片不显示
- 含图片的段落须单独设置为 **1.5倍行距**，不能使用固定行距
- python-docx 实现：直接操作 XML，在 `w:spacing` 元素上设置 `w:line="360"` + `w:lineRule="auto"`（不可用 `pf.line_spacing = None` 清空，否则无效）
- 图片段落同时设置：居中对齐，首行缩进为 0，段前段后间距为 0

### 4. 图标题与图注格式
- **图标题必须置于图片下方**（image 段落在前，caption 段落在后）
- 检测方法：若 `图N` 开头段落的下一段落含 `w:drawing`，则需交换
- python-docx 实现：`title_p._p.addprevious(image_p._p)` 将图片移至标题前
- **图标题字体字号与正文相同**：仿宋，三号（16pt），居中，固定行距 28.95pt，首行缩进 0
- 图注（注：...）字体：仿宋，五号（10.5pt），左对齐，固定行距 14pt

### 5. 统计指标斜体
- 统计学符号须使用斜体：*P*（含 P<0.05、P值）、*RR*、*OR*、*HR*、*CI*（含 95%CI）、*α*
- 实现：用正则拆分 run，对匹配部分设置 `<w:i/>` + `<w:iCs/>`，非匹配部分清除斜体
- 同时处理正文段落和表格单元格中的统计符号

### 6. 表格：标准学术三线表
- **边框**：顶线 1.5pt（`sz=12`），栏目分隔线 0.75pt（`sz=6`），底线 1.5pt；无纵线，无内部横线
- **填充**：移除所有单元格 `w:shd` 着色，不使用交替行色
- **字体**：仿宋，五号（10.5pt），固定行距 14pt，无首行缩进
- **表格标题**（表N...）：仿宋，三号（16pt），居中，固定行距 28.95pt，与正文字号相同
- **表注**（注：...）：仿宋，五号（10.5pt），固定行距 14pt
- **显著性行加粗**：统计检验 *P*<0.05 的数据行整行加粗；不使用颜色；标题行同样加粗
- **列宽自适应（字体度量法）**：根据各列最长内容的实际渲染宽度计算，总宽不超过页面内容宽（A4 = 156mm）
  - 算法：CJK 字符宽度 = font_pt，ASCII 字符宽度 = font_pt × 0.6，转换为 mm；每列加 4mm 内边距
  - 缩放规则：总需宽 ≤ 156mm 则等比放大至满宽；≤ 171.6mm（1.1×）则保持原比例；超出则等比缩小
  - python-docx 实现：同时更新 `tblPr/w:tblW` 和每个单元格的 `tcPr/w:tcW`（DXA单位）
  - 示例代码：
    ```python
    PT_PER_MM = 72 / 25.4
    def _line_width_mm(text, font_pt=10.5):
        pt = sum(font_pt if unicodedata.east_asian_width(c) in ('W','F') else font_pt*0.6 for c in text)
        return pt / PT_PER_MM
    def auto_col_widths(table, content_mm=156, font_pt=10.5):
        col_need = [max(_line_width_mm(line, font_pt) for row in table.rows
                        for line in row.cells[ci].text.split('\n')) + 4.0
                    for ci in range(len(table.columns))]
        total = sum(col_need)
        scale = content_mm/total if total<=content_mm else (1.0 if total<=content_mm*1.1 else content_mm*1.1/total)
        return [w*scale for w in col_need]
    ```

### 7. 文首信息移至落款
- 调查时间、报告日期、调查单位等信息不应置于标题下方
- 应将**报告日期**和**调查单位**移至正文末尾作为落款，删除调查时间行
- 落款格式：仿宋三号，右对齐，`right_indent = Pt(64)`（向左4个字符）

### 8. 正标题分行规则
- 标题过长一行放不下时，按**语义完整性**手动分行，不依赖自动换行
- 断点选在语义单元边界（如事件名称与文种之间）
- python-docx 实现：在 run 的 XML 中插入 `<w:br>` 元素实现软回车分行

## 自动排版脚本

本 skill 附带 Python 脚本 `scripts/gongwen_layout_v2.py`，可将任意 docx 文件按公文格式自动排版输出。

### 环境依赖

```bash
pip install lxml
```

### 基本用法

```bash
python3 scripts/gongwen_layout_v2.py <输入文件.docx> [输出文件.docx]
```

- **输入文件**：`src_docx` — 原始 docx（内容不限，脚本会按公文结构重新生成正文）
- **输出文件**（可选）：默认覆盖输入文件

### 输出格式说明

| 要素 | 格式 |
|------|------|
| 文件标题 | 方正小标宋简体·二号·居中 |
| 正文 | 仿宋三号·固定28.95pt·首行缩进2字符·两端对齐 |
| 一级标题（一、二、三……） | 黑体三号·段前段后0磅·左缩进2字符 |
| 二级标题 | 楷体三号·左缩进2字符 |
| 参考文献条目 | 仿宋三号·首行缩进2字符·两端对齐 |
| 落款 | 仿宋三号·右对齐·向右缩进4字符 |
| 页边距 | 上/下36mm，左/右27mm |
| 页眉/页脚 | 15mm / 27mm |

### 注意事项

- 脚本会**覆盖**原始 docx 中的所有正文内容，按固定结构重建
- 使用前建议备份原文件
- 脚本读取原始文件中的段落文本，按顺序映射到对应章节；如需调整内容结构，请直接修改脚本中 `process()` 函数里的正文构建代码
